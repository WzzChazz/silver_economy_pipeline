# 项目流转（最新 · 2026-06-16）

银发矩阵 · Silver Economy Pipeline —— 面向微信视频号「老照片·家庭回忆」的短视频自动化生产流水线。

> 战略定位：视频号是**获客漏斗**（养精准粉 + 转发裂变），变现端是《时光相册》小程序
> （用户上传自家老照片生成视频）。两者并行开发，起号期视频号**只做纯情感、不在口播露出引流**。

---

## 一、数据流闭环（换轴后）

```
①素材生成        ②评分入库            ③选题改写              ④数据回传            ⑤飞轮反哺
generate    →   story_engine    →    video_engine     →   performance_api  →   theme_performance
(唤起型)        换轴五维评分          短文案+结构轮换         /channels_rpa        + few-shot选样
                pain=转发力           +钩子动态控密度         回传播放/赞/转        (按互动综合分,
                resonance=唤起度      +发布助手落地钩子                            非完播率)
                  ↑                                                                  │
                  └──────────────── theme_weight 影响下一轮选题排序 ←─────────────────┘
```

**换轴要点：**
- 入库门槛看「转发驱动力 + 照片唤起度」，不再看「真实性」。
- few-shot 范本按「互动综合分（转发×3+评论×2+赞）」选，不再按完播率 →
  系统学的是「怎么让人转」，会拿 6058 那种高转发样本当老师。

## 二、命令清单

```bash
# —— 初始化 ——
python scripts/main.py setup        # 建库 + 3 个示例账号（新主题体系）

# —— 内容生产（推荐路径）——
python scripts/main.py generate [N] # 生成唤起型素材入库（每主题 N 条，默认 3）★新增，替代采集
python scripts/main.py health       # 看故事库健康（总量/可用/主题分布）
python scripts/main.py produce      # 为活跃账号各生成一条视频 + publish_guide.txt

# —— 数据闭环 ——
python scripts/main.py report       # 选题效果报告
python scripts/main.py scrape       # 本地 RPA 抓视频号数据回写
python scripts/main.py schedule     # 调度器（每日采集 + 数据回传）

# —— 旧链路（保留，可选）——
python scripts/main.py crawl        # 多渠道采集（知乎/小红书/微信），洗稿风险较高
```

## 三、单条视频的生成逻辑（produce 内部）

```
1. 拉素材   get_story_for_persona(软匹配: persona+theme → theme → 全库)
2. 定身份   _account_identity: 旁白性别/系列名/集数（按账号稳定）
3. 选结构   _pick_structure_hint: 按集数轮换 7 种叙事结构（防连撞）
4. 写脚本   _SCRIPT_PROMPT → LLM:
              · 15-25 秒 / 60-90 字
              · 第N种叙事结构开头（非永远"你家是不是也有"）
              · 共鸣 → 轻遗憾（起号期止于此）
              · 钩子库动态挑 0-2 个（互动/追更/转发）
              · 输出 hooks_used / interaction_hook / follow_hook / comment_hook
5. 配音     generate_audio（edge-tts，待换 F5-TTS 真人克隆音）
6. 画面     image_prompt → CogView 现画（待换真实老照片素材）→ 兜底本地/Unsplash
7. 合成     create_video：FFmpeg + Ken Burns + 反同质化滤镜 + ASS 字幕
8. 封面     generate_ab_covers：A/B/C 三版
9. 落地     publish_guide.txt：口播 + 封面 + 带头评论 + 互动/追更钩子（手动发布剧本）
10. 记录    record_production → 进飞轮
```

## 四、阶段化运营（MONETIZE_STAGE）

| 阶段 | 触发 | 内容形态 | 引流 |
|---|---|---|---|
| **0 起号期**（当前） | 粉丝<1k、未起量 | 纯情感，几乎内容号 | 零引流，口播+评论区都不提产品 |
| **1 起量期** | 单条破万、有转发 | 情感为主，偶露相册演示 | 评论区软承接一句 |
| **2 变现期** | 稳定起量、有咨询 | 情感+演示穿插 | 明确承接下单 |

> 起号期核心目标：**把转发率做起来 + 养精准粉**。变现出口（小程序）平行开发，等池子够大再放开引流。

## 五、真跑前置条件

| 命令 | 需要 | 本机现状 |
|---|---|---|
| `setup` / `health` / `report` | sqlite（自带） | ✅ 可跑 |
| `generate` | 有效 LLM key（httpx 已装） | ⚠️ key 401，换有效 key 即可 |
| `produce` | 上述 + `ffmpeg` + `edge_tts` + `whisper` + `torch` | ❌ 缺依赖 |
| `crawl` / `scrape` | `playwright` + `playwright install chromium` | ❌ 缺依赖 |

**最短验证路径**：配好有效 key → `generate 2` → `health` 看素材质量（最便宜、最关键的一步，不需 ffmpeg）。

## 六、下一步路线（已与运营讨论确定）

1. **真人克隆音**：F5-TTS 接入 `generate_audio` + 建参考音库（提升最大）。
2. **画面去假**：真实年代老照片素材库取代 CogView 现画；与《时光相册》样片打通。
3. **变现联动**：起量后开 `MONETIZE_STAGE`，评论区承接小程序流量。
4. **人设定死**：account_matrix 加 gender 字段，保证系列连续性 / 涨粉。

> 总原则：**先手动验证出 1 个能复制的爆款公式（看转发率），再让机器规模化复制。**
> 不在「还没验证一条能起量」时急着堆自动化矩阵。

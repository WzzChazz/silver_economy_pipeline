# 内容策略重构说明（2026-06-16）

> 一句话：把整条内容流水线的指挥棒，从**完播率思维**换成**转发率思维**——
> 因为真实数据证明，本赛道（中老年家庭情感）是「转发驱动社交裂变」，不是「完播进推荐池」。

---

## 1. 为什么改（数据复盘）

两条真实视频数据：

| 指标 | 视频① 老照片长文案 | 视频② 短·正能量·健康带给家人 |
|---|---|---|
| 播放 | 354 | **6058** |
| 完播率 | 17.23% | 20.82% |
| 点赞/评论/转发 | 0 / 0 / 0 | 17 / 23 / **10** |
| 涨粉 | 0 | 1 |

**结论：**
- 6058 那条 **推荐=0**，完全靠**转发/分享**滚出来 → 命门是转发率，不是完播率。
- 最用心的「老照片长文案」反而扑了 → 旧策略（30-45 秒长文案 + 回忆向 + 强完播钩子）押错了 KPI。
- 6000 人看了只涨 1 粉 → **缺「让用户不得不关注」的触点**（互动/追更/主页承接全空）。

旧重构（commit `185d996`）防限流做对了，但**进攻端用了通用短视频的「完播率方法论」，套错了赛道**。本次即「KPI 换轴」。

---

## 2. 改了什么（4 个代码文件 + 本地 .env）

### 2.1 `src/engines/video_engine.py`（内容生成核心）

- **`_SCRIPT_PROMPT` 全面重写：**
  - 目标换轴：从「讲催泪故事」→「让用户**对号入座**想起自家一张老照片」（= 想转发 + 想自己做，变现种子）。
  - 长度：110-150 字 / 30-45 秒 → **60-90 字 / 15-25 秒**（数据证 17-22 秒就划走）。
  - **叙事结构库（防同质化）**：内置 7 种开头/结构（唤起/悬念物件/数字反差/第一人称自述/今昔对比/提问代入/纯白描），
    由代码按集数**确定性轮换**（`_pick_structure_hint`），强制连续多条不撞同一种开头——LLM 单条调用无法自知上一条，必须代码层控。
  - **钩子库 + 动态匹配 + 控密度**：互动钩子（报年份/报数字）、追更钩子（主题合集+集齐心理）、转发钩子，
    由 LLM 按内容**最多挑 1-2 个**，纯情绪片可 0 钩子；明确禁止三件套全塞（雷同=被判模板号限流）。
  - **限流红线收紧**：所有结构（尤其悬念型/自述型）禁止「去世/走了/病了/住院/最后一面」等死亡卖惨字眼。
  - **变现埋线（起号期）**：正文埋「照片泛黄模糊」的轻遗憾，**只到痛点为止**，口播禁提「修复/小程序/教程」。
  - `image_prompt` 主体从「现画中国老人脸」→「一张老照片/老物件」，弱化人脸、减少 AI 假感。
  - 新增 JSON 输出字段：`hooks_used`（核对钩子密度）、`interaction_hook`、`follow_hook`。
- **`_fallback_script()`** 同步成短文案 + 第二人称 + 新字段（LLM 失败走兜底也不破功）。
- **新增 `_write_publish_guide()`**：把口播/封面/互动钩子/追更钩子/**小号带头评论**聚合成 `publish_guide.txt`，
  让 LLM 生成的钩子真正落到手上（手动发布时照此「剧本」操作）。
- `main()` 落盘改用发布助手；终端日志打印钩子密度/互动/追更/带头评论。

### 2.2 `src/engines/story_engine.py`（素材入口）

- **`_SCORE_PROMPT` 评分换轴**（JSON 字段名不变，避免破坏 DB 的 GENERATED score 列；只重定义语义）：
  - `score_pain` → **转发驱动力**（看完想不想转给家人）
  - `score_resonance` → **照片唤起度**（多少人想起自家一张照片）★最关键
  - `score_truth` → 年代真实质感；`score_freshness` → 年代锚点具体度；`score_rewrite` → 15-25 秒短视频改写潜力
- **入库阈值换轴**：`总分≥75 且 pain≥13 且 resonance≥13`（原为 pain+truth）。
- **新增 `generate_evocative_stories()`**：围绕主题 + 年代物件锚点池（的确良/二八自行车/老挂历…）**生成**
  「一代人共同经历」的唤起型素材，逐条评分入库。**零侵权、强对号入座，取代爬别人故事洗稿。**
- **`evaluate_story` 新增 `force_theme`**（真跑暴露并修复）：生成路径锁定主题——否则评分 LLM 会把唤起型素材
  （老物件/年代感）几乎全判成「年代记忆」，导致其它主题账号拉不到货、定位糊掉。

### 2.3 `src/core/database.py`（数据层）

- **`get_top_performing_scripts` 飞轮选样换轴**：few-shot 从「完播率最高」→ **互动综合分（转发×3 + 评论×2 + 赞）**，
  并去掉 `watch_rate>0` 过滤（否则 RPA 抓不到完播率的真实爆款全被漏掉）。
- **`get_story_for_persona` 软匹配**：`(persona+theme) → (仅 theme) → (全库)` 逐级放宽，
  保证冷启动时库里有货就一定拉得到，不因 persona 对不上就退回兜底。

### 2.4 `scripts/main.py`（CLI）

- 新增 **`generate`** 命令：`python scripts/main.py generate [每主题条数]`。
- **`cmd_setup` 示例账号换新体系**：`恩爱老伴/父母爱情`、`怀旧老人/年代记忆`、`孝心儿女/儿女孝心`
  （去掉踩红线的「独居/空巢」，theme 从废弃的 life/children/partner 改为新主题）。

### 2.5 `.env`（本地，不进版本库，已确认 gitignore）

- 填入 LLM key；`MONETIZE_STAGE=0`（起号期纯情感不引流）。
